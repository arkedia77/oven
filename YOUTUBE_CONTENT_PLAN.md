**채널**: LeoMusic (기존) 또는 새 채널 (MusicScore AI)

## 📺 시리즈 개요

### 콘셉트
```
프로 뮤지션들과 함께
"AI 음악 엔진을 어떻게 만드는가"를 공개하는 시리즈

→ 기술 + 음악 + 휴먼 스토리 결합
→ 뮤지션들의 노하우를 AI로 구현하는 과정
```

### 타겟 오디언스
- 음악 제작자 (음악 AI 학습)
- 피아니스트/기타리스트 (협업 기회)
- 음악 팬 (뮤지션 인터뷰)
- 엔지니어/개발자 (기술 디테일)

### 핵심 메시지
```
"AI는 음악가의 스타일을 배울 수 있다"
"프로 뮤지션의 표현력을 데이터화하는 방법"
"MIDI 레코딩 → AI 학습 → 음악 생성의 전 과정"
```

## 🎬 에피소드 구성 (총 12부 시즌 1)

### **에피소드 1-4: 피아노 편**

#### EP1: "피아니스트를 만나다" (20분)
**공개**: 2026-03-15

**내용**:
- 오프닝: Liszt 엔진이란?
- 피아니스트 소개 (신상, 경력, 특징)
- 인터뷰: 루바토란? (정의 + 실제 연주)
  - "같은 곡을 두 가지 방식으로" 라이브 연주
- 벨로시티와 다이나믹
- 페달 기법 데모
- 클로징: "다음 주, MIDI로 기록한다"

**자막/그래픽**:
- 피아니스트 기본 정보
- 루바토/벨로시티/페달 용어 설명
- 악보 시각화 (악보 이미지)

**음악**:
- BGM: 클래식 피아노 (저작권 프리)
- 인터뷰 중 실제 연주 음성

---

#### EP2: "MIDI로 연주를 기록하다" (18분)
**공개**: 2026-03-22

**내용**:
- MIDI 레코딩의 필요성
- 필요한 장비 (디지털 피아노, MIDI 인터페이스, DAW)
- 실제 세팅 과정 (5분)
- "같은 곡 3가지 스타일" MIDI 레코딩 라이브
  1. 클래식 스타일
  2. 팝 스타일
  3. 피아니스트 자유 스타일
- MIDI 파일 체크 (음성/벨로시티 확인)
- 데이터 정규화 과정 스크린샷

**자막/그래픽**:
- 필요 장비 리스트
- MIDI 채널 구조도
- 벨로시티 값 실시간 표시 (DAW 화면)
- 온타임 변화 시각화

---

#### EP3: "데이터로 보는 연주자의 개성" (22분)
**공개**: 2026-03-29

**내용**:
- 수집된 MIDI 데이터 분석
- 루바토 시각화
  - 타임라인에서 온타임 변화 그래프
  - "느린 섹션은 루바토가 크다"
- 벨로시티 분포
  - 히스토그램: 멜로디 vs 반주 강약 차이
  - "멜로디는 100~120, 반주는 40~70"
- 페달 패턴
  - CC64 신호 타임라인
  - "화음 바뀔 때마다 교체하는 패턴"
- 아티큘레이션 분석
  - 음 길이(duration) 분포
  - "레가토 비율: 80%"

**자막/그래픽**:
- 그래프, 히스토그램 (Python matplotlib)
- 실제 MIDI 데이터 스크린샷
- 수치로 표현된 "연주자 프로필"

**호스트 톤**:
- "이 숫자들이 바로 그의 음악이다"

---

#### EP4: "AI가 연주자의 스타일을 배우다" (25분)
**공개**: 2026-04-05

**내용**:
- Aria 모델 소개 (EleutherAI, MIDI Transformer)
- 파인튜닝의 개념
  - 사전학습된 모델 + 피아니스트 MIDI 데이터
- PyTorch 학습 과정 (스크린샷)
  - Loss 그래프
  - 학습 진행률
- AI 생성 테스트
  - 같은 악보, 다양한 스타일 생성
  - 피아니스트 스타일로 생성된 MIDI 청취
- 실제 연주 vs AI 생성 비교
  - 사이드바이사이드 (DAW 두 트랙)
  - "루바토가 비슷하네!"
- 피아니스트 피드백
  - "이 부분은 정확한데, 여기는 달라"

**자막/그래픽**:
- Transformer 아키텍처 다이어그램
- MIDI 토큰화 시각화
- Loss 함수 그래프
- 생성 결과물 오디오 스펙트로그램
