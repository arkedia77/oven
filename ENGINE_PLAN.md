## 0. 엔진 이름: **Liszt**

- Leo 선정 (2026-03-08)

---

## 1. 비전

**사람처럼 연주하는 AI 피아노** — 연주자 고유의 표현(버릇, 다이나믹, 루바토)을 살린 MIDI를 생성하고, 고품질 VST로 렌더링하는 방식.

Suno처럼 오디오를 통째로 생성하는 방식이 아닌, **MIDI 생성 → VST 렌더링** 파이프라인. 피아노 솔로 엔진을 먼저 완성하고, 이후 다른 악기로 확장.

---

## 2. Suno와의 차이

| | Suno | Leo 엔진 |
|--|------|----------|
| 출력 | 오디오 (분리 불가) | MIDI (파트 분리, 편집 가능) |
| 음색 | 모델 내장 (고정) | VST 교체 가능 (키스케이프 등) |
| 편집 | 불가 | 파트별 수정 가능 |
| 표현력 | 평균적 | 연주자 스타일 모델링 |
| 협업 | 불가 | 뮤지션과 함께 작업 가능 |

---

## 3. 퍼포먼스 모델링 (핵심 차별점)

같은 곡도 연주자마다 다른 이유:
- **루바토**: 박자를 미세하게 당기고 늘리는 습관
- **벨로시티 패턴**: 멜로디 강조, 반주 억제 비율
- **페달링**: 밟는 타이밍과 길이
- **아티큘레이션**: 레가토/스타카토 성향
- **앞꾸밈음/트릴 처리** 방식

→ 이 모든 것이 MIDI에 담겨있음. **퍼포먼스 MIDI 학습 = 연주자 개성 학습.**

```
악보 MIDI (what to play)
        +
퍼포먼스 스타일 조건 (how to play: 연주자 A / B / C)
        ↓
표현이 살아있는 MIDI 출력 → 키스케이프 렌더링
```

## 4. 기술 스택

### 모델 (피아노)
- **베이스**: Aria (EleutherAI 오픈소스, ARIA-MIDI 118만곡 학습됨) ← 피아노 특화
- **파인튜닝**: Leo 셀렉 데이터 + MAESTRO + ATEPP 으로 퍼포먼스 표현 강화
- **구조**: MIDI 토큰 시퀀스 Transformer (LLM과 동일 아키텍처, 도메인만 다름)
- 처음부터 학습 X → Aria 위에 쌓는 방식

### 렌더링 + 후처리
- MIDI 출력 → **Pedalboard** (Spotify 오픈소스, Python VST 호스트)
- 피아노 음색: 키스케이프 (Leo 보유)
- DAW 불필요, 완전 자동화 파이프라인

**후처리 체인 (피아노 전용, 절제 원칙):**
```
키스케이프 VST 렌더링
    ↓
EQ
  — 80Hz 이하 컷 (울림 정리)
  — 200~400Hz 미세 컷 (뭉침 제거)
  — 5kHz+ 약간 부스트 (선명도)
  — 2~4kHz 조정 (타건감)
    ↓
Compressor (ratio 2:1~3:1, 약하게)
  — 자연스러운 다이나믹 유지
  — 강하게 쓰면 피아노 표현력 죽음
    ↓
Reverb (장르별 프리셋)
  — 클래식: 홀 리버브 (긴 꼬리)
  — 팝/발라드: 룸 리버브 (짧고 자연스럽게)
  — 트로트: 별도 연구 필요
    ↓
Limiter (-0.3dBFS, 마지막 한 번만)
    ↓
완성 오디오
```
- 장르별 프리셋을 JSON으로 관리 → 쉽게 수정/추가

### 인프라
- GPU: RunPod (기존 ACE-Step 인프라 활용)
- 데이터: `/Volumes/data/score/`

### 설치된 도구
- `demucs` — 오디오 소스 분리 (보컬/악기)
- `basic-pitch` — 오디오 → MIDI 변환
- `pedalboard` — Python VST 호스팅
- `pretty_midi` — MIDI 분석/가공

## 6. 전체 로드맵

### Phase 1: 인프라 (현재 진행)
- [x] 데이터 수집 (400만곡+)
- [ ] DB 구축 + 중복 제거 + 품질 필터링
- [ ] Leo 셀렉 곡 태깅 (quality_score, genre, use_for_training)

### Phase 2: 피아노 솔로 엔진 ← 첫 목표
- 목표: 사람처럼 연주하는 피아노 MIDI 생성
- 베이스 모델: Aria (EleutherAI)
- 파인튜닝 데이터: ARIA-MIDI + MAESTRO + ATEPP + PDMX 피아노
- 렌더링: 키스케이프 → 오디오 출력
- 검증: Leo가 직접 듣고 퀄리티 평가
- **피아노 하나 제대로 → 나머지 확장은 구조가 이미 갖춰짐**

### Phase 3: 밴드 악기 확장
- 피아노 엔진 구조 재활용
- 기타 / 베이스 / 드럼 / 키보드 순차 추가
- 각 파트 독립 MIDI 생성 → VST 렌더링 → 합산

### Phase 4: 뮤지션 협업 툴
- Leo 수노 ~3,000곡 중 셀렉 → AI 커버/재편곡
- 멜로디 스케치 → AI 편곡 → 파트별 수정
- Leo 취향/스타일 개인화 학습

### Phase 5: 트로트 전문 모델 ★
- 프로 작곡가/가수 협업 (이미 협의됨)
- 꺾기, 바이브레이토, 구전 표현 캡처
- MIDI 퍼포먼스 레코딩 방식 별도 설계
- 전 세계 블루오션 — 일반 모델과 완전 분리 학습

### Phase 6: 악기 연습 서비스
- MXL 악보 → OSMD 렌더링
- MIDI 재생 + 악보 커서 연동
- 난이도 분류, 연주자 스타일 모범 연주 시연

### Phase 7: 보컬 엔진 (장기)
- 가장 난이도 높음
- 전략: **수노를 경쟁자가 아닌 데이터 생성기로 활용**
  - 수노 곡 → Demucs로 보컬 분리 → Basic Pitch로 MIDI화
  - 수노 품질 올라갈수록 학습 데이터 자동 상승
- Leo 수노 ~3,000곡 = 보컬 학습 시드
- 트로트 보컬은 Phase 5 프로 협업과 연계
